# 🚀 Multi-Platform Chat Aggregator - Plesk Kurulum Rehberi\n\n## 📋 Özellikler\n- **3 Platform Desteği**: YouTube, Twitch, Kick chat'lerini tek sayfada birleştirme\n- **Real-time Updates**: Canlı mesaj güncellemeleri\n- **Platform Filtreleme**: Her platformu ayrı ayrı görüntüleme\n- **Responsive Tasarım**: Mobil uyumlu arayüz\n- **Dark/Light Mode**: Tema değiştirme özelliği\n- **Memory Optimized**: Düşük RAM kullanımı (~200MB)\n\n## 📦 Plesk Sunucusuna Kurulum\n\n### 1. Dosyaları Upload Etme\n1. **Plesk Panel**'e giriş yapın\n2. **Dosya Yöneticisi**'ne gidin\n3. **httpdocs** klasörüne gidin\n4. **plesk-upload.zip** dosyasını upload edin\n5. ZIP dosyasını **Extract** (çıkart) edin\n6. `dist` klasörü içindeki tüm dosyaları `httpdocs` ana dizinine taşıyın\n\n### 2. Dosya Yapısı (httpdocs içinde)\n```\nhttpdocs/\n├── index.html          # Ana sayfa\n├── assets/             # CSS, JS ve font dosyaları\n│   ├── index-*.css\n│   ├── index-*.js\n│   ├── vendor-*.js\n│   ├── services-*.js\n│   └── remixicon-*.*\n```\n\n### 3. YouTube API Anahtarı (Opsiyonel)\nYouTube chat'i için API anahtarı gereklidir:\n\n1. [Google Cloud Console](https://console.cloud.google.com/) açın\n2. Yeni proje oluşturun veya mevcut projeyi seçin\n3. **APIs & Services > Library** bölümüne gidin\n4. **YouTube Data API v3** aratın ve etkinleştirin\n5. **APIs & Services > Credentials** bölümüne gidin\n6. **Create Credentials > API Key** seçin\n7. API anahtarını kopyalayın\n\n**API Anahtarını Ekleme:**\n- `index.html` dosyasını düzenleyin\n- `your_youtube_api_key_here` yazan yeri gerçek API key ile değiştirin\n\n## 🎯 Kullanım\n\n### URL Parametreleri ile Otomatik Başlatma\n```\nhttps://yourdomain.com/index.html?twitch=kanaladi&kick=kanaladi&youtube_vid=videoid&theme=dark\n```\n\n**Parametreler:**\n- `twitch=kanaladi` - Twitch kanal adı\n- `kick=kanaladi` - Kick kanal adı  \n- `youtube_vid=videoid` - YouTube canlı yayın video ID'si\n- `theme=dark` veya `theme=light` - Tema seçimi\n\n### Örnek URL'ler\n```\n# Twitch ve Kick için:\nhttps://yourdomain.com/?twitch=bertugfahriozer&kick=bertugfahriozer&theme=dark\n\n# Tüm platformlar için:\nhttps://yourdomain.com/?twitch=summit1g&kick=adin&youtube_vid=dQw4w9WgXcQ&theme=light\n```\n\n### Manuel Kullanım\n1. Web sitesini açın\n2. Üst kısımdaki input alanlarına kanal adlarını girin:\n   - **Twitch**: Kanal adı (örn: summit1g)\n   - **YouTube**: Video ID (URL'den alın)\n   - **Kick**: Kanal adı (örn: adin)\n3. **\"Bağlan\"** butonuna tıklayın\n4. Chat mesajları real-time olarak görünmeye başlayacak\n\n## ⚙️ Konfigürasyon\n\n### YouTube Video ID Nasıl Bulunur?\nYouTube canlı yayın URL'i:\n```\nhttps://www.youtube.com/watch?v=dQw4w9WgXcQ\n```\nVideo ID: `dQw4w9WgXcQ` (v= sonrasındaki kısım)\n\n### Platform Durumu\n- **Twitch**: ✅ Tam çalışır (API key gerektirmez)\n- **YouTube**: ⚠️ API key gerektirir (günlük 10,000 istek limiti)\n- **Kick**: ✅ Tam çalışır (API key gerektirmez)\n\n## 🔧 Sorun Giderme\n\n### YouTube Mesajları Görünmüyor\n- API anahtarının doğru olduğundan emin olun\n- Video ID'nin canlı yayın olduğunu kontrol edin\n- Browser console'da hata mesajlarını kontrol edin\n\n### Twitch/Kick Mesajları Görünmüyor\n- Kanal adının doğru yazıldığından emin olun\n- Kanalın canlı yayında olduğunu kontrol edin\n- Internet bağlantınızı kontrol edin\n\n### Yüksek RAM Kullanımı\n- Sayfa sık sık yenileyin\n- Browser cache'ini temizleyin\n- Birden fazla sekme açmayın\n\n## 🎨 Özelleştirme\n\n### Tema Değiştirme\n- Sağ üst köşedeki ay/güneş ikonuna tıklayın\n- Veya URL'ye `?theme=dark` veya `?theme=light` ekleyin\n\n### Mesaj Limiti\nVarsayılan olarak son 50 mesaj gösterilir. Bu sınır performance için optimize edilmiştir.\n\n## 📊 Teknik Detaylar\n- **Framework**: Vue.js 3 + Vite\n- **Real-time**: WebSocket (Twitch), Polling (YouTube, Kick)\n- **Memory Usage**: ~200-300MB\n- **Browser Support**: Chrome, Firefox, Safari, Edge (modern versions)\n- **Mobile Friendly**: ✅ Responsive tasarım\n\n## 📞 Destek\nSorularınız için:\n- GitHub Issues: Repository'de issue açın\n- Email: Proje sahibi ile iletişime geçin\n\n## 🔒 Güvenlik\n- API anahtarları client-side'da saklanır\n- HTTPS kullanımı önerilir\n- API quotalarına dikkat edin\n\n---\n**Not**: Bu uygulama eğitim ve kişisel kullanım amaçlıdır. Ticari kullanım için platform API şartlarını kontrol edin.\n"
